## 一、工程概览

### 1.1 这是什么

一套基于 **STM32F407ZGT6 + F103C8T6 双芯片架构**的智能仪表盘系统，具备：

- 800×480 全彩 LCD 图形界面（LVGL 8.3）
- CAN 总线实时传感器数据采集与显示
- 双区 Bootloader/APP 架构，支持 CAN ISO-TP 远程固件升级
- SPI Flash (W25Q128) 文件系统存储图片与固件备份
- OV7670 摄像头实时画面预览

各层职责与约束:
- **APPLICATION**: 只调 SERVICE, 禁止直接调 DRIVER/HAL

- **DRIVER**: 调 HAL, 可被同层调用 (需注释说明理由)
- **HAL**: 最底层, 无项目依赖

例外（有意保留的跨层调用，因性能优化）:
- `ov7670.c`(DRIVER) → `nt35510.h`(DRIVER) : 摄像头直写 LCD 寄存器

## 四、关键技术实现

### 4.1 双区 OTA 升级

```
内存布局：
0x08000000 ┐ Bootloader (64KB)
           │   └─ 启动 → 参数扇区校验 → 检查 ota_flag
0x08010000 ┤           → flag=1: 等待 ISO-TP OTA
           │ APP (832KB)     → flag=0: HMAC 签名验证 → 跳转
0x080DFF80 ┤ signature.bin (40B: fw_size + HMAC + Magic)
0x080E0000 ┤ 参数扇区 (12B: magic + version + flag + checksum + count)
0x08100000 ┘
```

OTA 升级流程：
```
APP 运行中 → 用户按 OTA 按钮 / CAN 触发
  → 备份当前固件到 SPI Flash (firmware.bak)
  → 设 ota_flag=1 → NVIC_SystemReset()
  → Bootloader 启动 → 检测 flag=1
  → 通过 CAN ISO-TP 接收新固件 → 写入 Flash
  → 清除 flag → 跳转 APP
```

升级失败自动回滚：
```
Bootloader 检测到签名无效
  → 从 SPI Flash 读取 firmware.bak
  → 恢复到 APP 分区
  → 跳转 APP（保留旧版本运行）
```

### 4.2 OV7670 摄像头（GPIO FIFO 方案）

采用 GPIO 直接操作 AL422B FIFO 的时序方案，而非 DCMI 硬件接口：

```
VSYNC 中断驱动帧捕获：
  第 1 个 VSYNC（上升沿）→ 复位 FIFO 写指针 → 使能写入
  第 2 个 VSYNC（上升沿）→ 禁能写入 → 帧就绪
  camera_refresh() 读取 FIFO → 写入 LCD

EXTI9_5 中断处理（stm32f4xx_it.c）:
  - APP 链接：完整 FIFO 控制代码（#ifndef LFS_READONLY）
  - BL 链接：仅清除 PR 标志（空操作）
  - BL 入口处 NVIC 已关，PA8 浮空不触发
```

关键修复（4 轮迭代）：
1. **启动卡死** — BL 中 PA8 浮空 EXTI 触发循环
2. **画面全蓝** — 未拉低 FIFO CS 脚 (PG15)
3. **画面撕裂** — 在帧写入途中读取 FIFO
4. **底噪过多** — AGC 增益上限过高

### 4.3 CAN 通信协议

| ID | 方向 | 用途 | 格式 | 频率 |
|----|------|------|------|------|
| 0x12 | F103→F407 | 传感器数据 | [T+40][Tdec][H][Hdec][knob][key] 6B | 50Hz |
| 0x13 | F103→F407 | RTC 时间 | [y][m][d][h][min][s] 6B | 1Hz |
| 0x14 | F407→F103 | 时间同步 | [y][m][d][h][min][s][sum] 7B | 触发 |
| 0x0B1 | F103→F407 | ISO-TP OTA | 流式，首字节 0xBF | — |

### 4.5 图片加载三级状态机

`spi_img_loader.c` 实现三级故障降级架构，每次启动自动判断当前硬件状态并选择最优路径：

```
[正常] LFS → 外部 SRAM → LVGL
         条件: SRAM 读写正常 + LFS 文件齐全
         过程: LFS 文件逐一读取到 FSMC 外部 SRAM (0x68000000)
               lv_img_dsc_t.data 指向 SRAM 地址
         额外: 擦除内部 Flash 扇区9(备份区)，释放 ~128KB 给其他用途

[降级] LFS → 内部 Flash 扇区9 → LVGL
         条件: SRAM 故障 但 SPI Flash/LFS 正常
         过程: 从 LFS 逐文件拷贝到内部 Flash 扇区9 (0x080A0000, 128KB)
               lv_img_dsc_t.data 指向 Flash 地址，LVGL 直接从 Flash 读取像素
               backup_is_valid() 检测已有备份则跳过拷贝

[抢救] 编译后备 → LVGL
         条件: SRAM 和 SPI Flash 均故障
         过程: lv_img_dsc_t.data 指向内部 Flash 中编译的 ui_img_*.data
               LVGL 直接读取固件中的像素数据，不依赖任何外部存储
```

内部 Flash 布局（APP 区域）：

```
0x08000000 ┐ Bootloader (64KB)
0x08010000 │
           │ APP (544KB)
0x080952DC │
0x080A0000 ┤ 运行时备份区 (扇区9, 128KB) ← 降级时存放图片数据
           │  存储格式: [4B BACKUP_MAGIC] [图片数据...]
0x080C0000 ┤ 空闲
0x080DFF80 ┤ 签名块 (40B)
0x080E0000 ┤ 参数扇区 (128KB)
0x08100000 ┘
```

后备图片保留在固件中（编译期数据，共约 77KB），作为最后一道防线。

### 4.7 固件签名 (HMAC-SHA256)

- 算法：HMAC-SHA256，32 字节密钥
- 签名块位置：0x080DFF80（APP 末尾前 40B）
- 签名块结构：`[fw_size:4][HMAC:32][magic:4]`
- 验证时机：Bootloader 跳转前
- 开发模式：无签名时仅警告，不拦截

| 5 | **NT35510 花屏** | 颜色随机错乱 | 缺 SWRESET 命令 | 加 0x0100 + 120ms 等待 |
| 6 | **LVGL 动画卡死** | 切换后页面只绘一半 | `lv_scr_load_anim` 依赖定时器 | 全改 `lv_scr_load` |
| 7 | **画面全蓝** | 摄像头区域蓝色块 | CS(PG15) 未拉低 | `OV7670_CS = 0` |

#### WiFi OTA 双模式

```
开发模式 (定向推送):
  PC ──TCP:8080──► ESP8266 ──USART3──► F407
    1. WHO → MCU 回复 96-bit UID (握手校验)
    2. OTA:size → 写入 SPI Flash 备份区 (0xE00000)
    3. NVIC_SystemReset() → Bootloader 检测恢复

量产模式 (广播推送):
  PC ──UDP:8081──► 所有 ESP8266 (OTA_N:size:PC_IP:port)
    各板 ──TCP:8765──► PC 独立下载固件
    Bootloader 检测 SPI_BAK_MAGIC → 秒级恢复 → 跳转新 APP
```

#### OTA 流时序

```
APP 运行 → WiFi 连接 → TCP Server :8080 + UDP 监听 :8081
  │
  ├─ [定向] PC 连接 :8080 → WHO/UID → OTA:size
  │                              ↓
  │              接收 +IPD 帧 → 写入 SPI Flash → 备份头 → 复位
  │
  ├─ [广播] PC → UDP :8081 → OTA_N:size:server:port
  │                              ↓
  │              各板连 TCP server → 下载 → 写入 → 备份头 → 复位
  │
  └─ Bootloader 启动 → ota_flag=1 → SPI_BAK_MAGIC 匹配?
       ├─ yes → EN25Q128_RestoreFirmware() → 签名验证 → 跳转新 APP
       └─ no  → CAN ISO-TP 等待 (60s 超时 → 尝试 SPI Flash 恢复)
```
