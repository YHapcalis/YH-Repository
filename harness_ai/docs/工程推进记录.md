## 二、核心能力

### 2.1 四个后端

| 后端 | 启动方式 | 适用场景 |
|------|---------|---------|
| `--backend swd` | OpenOCD TCL (6666) | 真硬件 SWD 读写 |
| `--backend renode` | Renode TCP (12347) | 全仿真 (LCD 可读像素) |
| `--backend hybrid` | Renode + SWD 自动路由 | LCD 走 Renode, CAN 走 SWD |
| 后台自动启动 | `_ensure_monitor_ready()` | 烧录后自动启动 OpenOCD + reset run |

### 2.2 AI 决策

- 模型: `deepseek-v4-flash` (通过 DeepSeek Anthropic 兼容接口)
- thinking budget: 32000 tokens
- max_tokens: 48000
- timeout: 600s
- tool_choice: `{"type": "any"}` (DeepSeek thinking mode 兼容)

### 2.3 SWD 页面导航 (2026-07-11 实机验证)

调用方式: `halt → resume [link_address]` (不可用 `reg pc + resume`)

```
正确翻页顺序:
  main → mode → main → settings → main → clock → main → camera → main
  每次切换后等待 ≥5s (LVGL 渲染 + SPI Flash 读图 + 动画)

地址格式: .map 文件链接地址 (不加1, resume 自动设 Thumb 位)
LR: 由 resume 保留, 函数返回自动回到 RTOS
```

已验证通过的函数地址:

| 函数 | map 地址 | 参数 |
|------|---------|------|
| `mode_ui_show/hide` | 0x08043088 / 0x080430CC | 无 |
| `settings_ui_show/hide` | 0x080434C0 / 0x080434E8 | 无 |
| `clock_ui_show/hide` | 0x08044648 / 0x080445B4 | 无 |
| `camera_ui_show/hide` | 0x080448E8 / 0x0804499C | 无 |
| `app_ui_update_sensor` | 0x08042370 | r0=temp(float), r1=hum(float), r2=knob, r3=key_id |

### 2.4 断言引擎

| 检查类型 | 说明 |
|---------|------|
| frequency | 信号翻转频率 (LED/PWM) |
| range | 值在 [min, max] 区间 |
| monotonic | 单调递增/递减 |
| change_detected | 变量在观察期内变化 |
| pattern | 序列匹配 |
| state_machine | 状态机跳转顺序 |
| stable_after | 后期趋于稳定 (PID) |
| rate | 变化率 (增量/秒) |

支持 `type: float` (IEEE754 自动转换) 和 `width: 8|16|32` (读取位宽)。

### 2.5 策略门控

- 文件白名单 (按场景限制可修改的文件)
- 内容安全 (禁止 RCC/Flash/NVIC 等危险模式)
- ELF 符号检查 (编译后验证关键参数)
- 审计日志 + 连续违规检测

### 2.6 分层记忆

- 短期记忆: JSONL 文件, 按天分片, 最近 2 小时
- 长期记忆: SQLite, 跨场景模式发现
- 自动注入 AI 提示上下文

### 2.7 熔断降级

熔断器状态机: CLOSED → 3次失败 → OPEN → 冷却30s → HALF_OPEN → 成功1次 → CLOSED

模式自动切换: hybrid → renode_only / swd_only → degraded → 自动恢复

## 三、关键的踩坑记录

### 3.1 函数调用: `resume [map_addr]` 而非 `reg pc + resume`

2026-07-11 实机验证:

```
❌ halt → reg pc 0x08043089 → reg lr 0x080FFFFE → resume  → HardFault
✅ halt → resume 0x08043088 → sleep 5000 → halt          → 8/8 通过
```

根因: `resume` 命令忽略 `reg pc` 的修改, 必须用 `resume [addr]` 直接指定入口。

### 3.2 页面切换必须经过主页

```
❌ mode → settings  (子页间直跳 → LVGL 资源冲突 → 卡死)
✅ main → mode → main → settings → main → ...
```

### 3.3 LVGL 渲染不能被 halt 打断

```
❌ resume → sleep(0.5s) → halt → resume  (halt 冻住动画定时器)
✅ resume → sleep(5s)                    (不 halt, LVGL 自由渲染)
```

### 3.4 Windows 路径反斜杠

OpenOCD 和 GDB 都不认 Windows 反斜杠 `\b` 被解析为退格符。
所有路径必须用正斜杠: `Path.as_posix()`。

### 3.5 Windows 终端 GBK 编码

Rich 输出的 Unicode 字符 (emoji) 在 GBK 终端崩溃。
解决方案: 所有输出用 ASCII-safe 字符, stdout 包装 `errors='replace'`。

### 3.6 CAN 总线变量被覆盖

`g_can_sensor` 由 F103 通过 CAN (50Hz) 写入, SWD 本地写 `knob`/`key_event` 会被下一帧 CAN 数据覆盖。页面导航不能用变量写, 必须用函数调用 (`resume [addr]`)。

## 四、待办 / 已知限制

| 事项 | 说明 | 优先级 |
|------|------|--------|
| LCD Renode 仿真 | NT35510.cs 模型可读像素, 但仿真速度慢需预热 | 低 |
| 摄像头物理激活 | `g_camera_active = 0` (硬件未连接或初始化失败) | 低 |
| CI/CD 集成 | 目前手动运行, 未接入自动化流水线 | 中 |
| 模型成本控制 | 目前固定 deepseek-v4-flash, 未做按任务路由 | 低 |
| 大规模并行子代理 | 串行执行子代理, 未做并行 pool | 低 |

## 五、快速命令参考

```powershell
# 全自动闭环 (AI 分析+修改代码+验证)
python embed_harness.py --backend swd --scenario scenarios/full_monitor.yaml --dir E:/.../MY_OTA_GUI --elf build/Debug/MY_OTA_GUI.elf --iterations 10 --skip-build

# 纯监控 (不修改代码, 不调用 AI)
python embed_harness.py --backend swd --scenario scenarios/full_monitor.yaml --dir ... --monitor-only --no-ai --export-dashboard ./traces

# 启动 Web Dashboard
python -m streamlit run harness_dashboard.py

# 页面导航测试 (手动)
python -c "
from monitor_client import OpenOCDClient, SWDInputSimulator
sim = SWDInputSimulator(OpenOCDClient())
sim.switch_page('mode')   # 进 Mode 页
sim.switch_page('main')   # 回主页
"
```
